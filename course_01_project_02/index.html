A Visual Explanation of Bubble Sort

Explanation

In the world of computer programming, one of the most standard problems one encounters, is the need to sort a set of elements. For example, if we have a list of values, [4, 2, 6, 7, 8, 5, 4], to sort it is to place these numbers in increasing order: [2, 4, 4, 5, 6, 7, 8]. This is a well-studied field, and there are numerous algorithms available for the task. One such algorithm is known as Bubble Sort, and we will be discussing that here.

One of the standard ways to measure performance of sorting algorithms is to think in terms of "swaps" (switching the positions of any two elements), and comparing how many swaps are needed on average to sort a random set of numbers.

Bubble sort works by systematically going through, and swapping adjacent elements if they're out of order, repeating the process as needed. This results in the largest elements "bubbling up" to the end of the list, which is where the name of the algorithm comes from. Below, you will find a snippet of pseudo-code, as well as a visual representation of how it works, courtesy of Wikipedia.

procedure bubbleSort( A : list of sortable items )
    n = length(A)
    repeat
       newn = 0
       for i = 1 to n-1 inclusive do
          if A[i-1] > A[i] then
             swap(A[i-1], A[i])
             newn = i
          end if
       end for
       n = newn
    until n = 0
end procedure

It's very important to note that while Bubble Sort is a neat way to teach sorting and introduce the concept of algorithms to students of computer science, it's actually a relatively inefficient way to go about it. Sorting algorithms used in practice are much more sophisticated. Still, there is some elegance to its simplicity.
